## Why `cmakefmt`?

<CardGrid stagger>
  <Card title="Zero dependencies" icon="rocket">
    One native Rust binary. No Python environment, no virtualenv bootstrap, no dependency
    drift. Drop it in CI and forget about it.
  </Card>
  <Card title="Workflow-first" icon="puzzle">
    `--check`, `--diff`, `--staged`, `--changed`, `--files-from`, and six machine-readable
    report formats. Every power-user feature `cmake-format` made you script around is built in.
  </Card>
  <Card title="Knows your commands" icon="open-book">
    Teach `cmakefmt` the argument structure of your project's custom CMake functions and
    macros — no more generic token wrapping for commands you wrote.
  </Card>
  <Card title="Errors that actually help" icon="warning">
    Parse and config failures include file, line, column, source context, and a reproduction
    hint — not an opaque wall of parser noise.
  </Card>
</CardGrid>

## Quick Start

Install in one line:

```bash
# macOS and Linux — no Rust toolchain needed
brew install cmakefmt/cmakefmt/cmakefmt

# Rust developers
cargo install cmakefmt-rust
```

Then:

```bash
cmakefmt --version
cmakefmt --dump-config > .cmakefmt.yaml   # generate a starter config
cmakefmt --check .                         # dry-run: show what would change
cmakefmt --in-place .                      # apply formatting
cmakefmt --staged --check                  # pre-commit: format only staged files
```

## Performance

| Metric | Value |
| --- | --- |
| Geometric-mean speedup vs `cmake-format` | **20.69×** |
| End-to-end `format_source`, large synthetic input (1000+ lines) | 8.82 ms |
| Serial whole-corpus batch (220 files) | 184.5 ms ± 1.3 ms |
| `--parallel 8` whole-corpus batch | **48.5 ms ± 1.5 ms** |
| Head-to-head on 612-file repo vs `cmake-format` (sequential) | **~97× faster** |

Fast enough for local dev, pre-commit hooks, editor integrations, and CI — all at once.
See [Performance](/performance/) for the full methodology and benchmark environment.
